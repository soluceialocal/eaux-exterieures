1. Contexte et objectifs

Les gestionnaires d'eaux extérieures (baignade, pêche, irrigation, écologie) manquent d'un outil simple, terrain, pour répondre à une question fondamentale : l'eau est-elle dans le même état qu'avant ?

Les outils existants (portails DCE, NAÏADES, Hub'Eau) sont conçus pour des techniciens OFB et ne permettent pas une lecture rapide de l'état courant par un non-spécialiste sur le terrain.

Proposition de valeur
Transformer les données physico-chimiques brutes de l'Agence Française pour la Biodiversité en un signal unique — Stable / Vigilance / Alerte — accessible hors-ligne sur téléphone, sans formation spécifique.

Objectifs mesurables

  • Détection précoce de dérive avant franchissement des seuils DCE réglementaires
  • Zéro jargon algorithmique visible dans la vue principale
  • Fonctionnement intégral hors-ligne (zones sans réseau)
  • Couverture nationale France métropolitaine — 5 types hydrologiques
  • Taux de faux positifs < 2 % sur séries rivières/fleuves (validé MTDC v3.3)

4. Exigences UX terrain

Principe directeur
L'application répond à une seule question : "Est-ce que l'eau est dans le même état qu'avant ?" Aucun paramètre algorithmique (Φ̃, phi1, G_min…) ne doit apparaître dans la vue principale.

Contraintes d'usage terrain

  • Lumière directe — contraste minimum 4.5:1 (WCAG AA), texte > 14px
  • Gants / grands doigts — zones cliquables minimum 44×44 px (Apple HIG)
  • 3G lent ou absent — chargement complet < 3s en 3G, fonctionnel offline
  • Batterie faible — pas d'animations continues sauf alertes critiques
  • Non-technicien — zéro sigle, acronyme ou code non expliqué en vue principale

Structure de navigation

  • Topbar : nom appli / recherche / date / compteurs statut
  • Barre secondaire : filtres usage + toggle Liste/Carte
  • Vue liste : cartes groupées Alertes → Vigilance → Stables, avec sparkline 30j
  • Vue carte : markers colorés OSM, popup avec bouton détail
  • Panel détail (3 onglets) : Terrain · Graphiques · Technique

6. Exigences PWA

Critères d'installation (Lighthouse PWA)

  • ✅ manifest.json valide avec icônes 192×192 et 512×512
  • ✅ Service Worker enregistré sur start_url
  • ✅ HTTPS (ou localhost pour tests)
  • ✅ theme-color défini (#2563eb)
  • ✅ display: standalone (plein écran sans barre navigateur)
  • ✅ apple-touch-icon pour iOS (Safari < 16.4)

Comportement offline attendu

  • Application démarre et s'affiche intégralement sans réseau
  • Données de la dernière synchronisation visibles
  • Bandeau "Mode hors-ligne" affiché clairement
  • Carte : tuiles déjà visitées disponibles, zones non-visitées = vide gris
  • Aucun crash, aucun écran blanc en l'absence de réseau

8. Moteur MTDC — intégration

Algorithme MTDC v3.3

MTDC (Méthode de Traçage des Dérives de Composition) détecte une dérive de l'état physico-chimique d'un cours d'eau avant le franchissement des seuils réglementaires DCE.

SymboleDéfinitionValeur type rivière
RsMédiane glissante (baseline saisonnière)n_base = 24 mois
Φ = X/RsRatio signal/baseline—
Φ̃Φ lissé (EWMA)—
G = dΦ̃/dtVitesse de dérive—
G_minSeuil adaptatif (p75 zone contrainte)~0.074
phi1Seuil d'amplitude auto-calibré~1.3465
TPersistance minimale avant alarme3 périodes

L'alarme est déclenchée quand Φ̃ > phi1 ET G > G_min pendant T périodes consécutives.

Classification hydrologique (5 types)

TypeModule m³/sn_baseT alarme
Ruisseau< 112 mois2 périodes
Rivière1–10024 mois3 périodes
Fleuve> 10036 mois3 périodes
Lac / retenue— (volume)6 ans3 périodes
Performance MTDC v3.3 (500 simulations)
Taux de faux positifs : 0 % rivière/fleuve, 1.2 % ruisseau. Temps de calcul : 0.64 ms pour 288 points (1 560 analyses/sec). G_min adaptatif (p75 zone contrainte structurante) — amélioration clé vs v3.2.

9. RGPD et sécurité

Données personnelles

Dans sa version V1, l'application ne collecte aucune donnée personnelle. Elle est en lecture seule, sans compte utilisateur, sans tracking, sans cookies tiers.

Données environnementales

  • Toutes les données sont issues de sources publiques sous Licence Ouverte 2.0 (Etalab)
  • Attribution obligatoire : "Données Hub'Eau / NAÏADES / OFB / Agences de l'eau"
  • Les données ne sont pas modifiées, seulement agrégées (médiane mensuelle)

Sécurité V2 (si saisie terrain)

  • HTTPS obligatoire pour l'enregistrement du service worker
  • Données terrain stockées en IndexedDB local (pas cloud par défaut)
  • Sync chiffrée HTTPS si backend ajouté
  • Aucun accès à la géolocalisation sans consentement explicite
